# SSH Tools 使用文档

`sshtools` 是一个面向多主机场景的命令行 SSH 管理工具，交互风格接近 JumpServer，支持主机分组、模糊搜索、交互式 SSH、SFTP 上传下载、私钥登录和默认公网/内网连接策略。

## 1. 功能概览

- 交互式命令行主菜单
- 按主机名、IP、全局 ID 快速连接
- 支持 `groups` 分组和 `p <tag>` 分组过滤
- 支持密码登录和 SSH 私钥登录
- 支持默认使用公网 IP 或内网 IP
- 支持文件上传、目录递归上传、文件/目录下载
- 支持配置热重载
- 支持内嵌演示配置兜底启动

## 2. 环境要求

- Go 1.22 或更高版本
- Linux / macOS 终端环境
- Windows 可使用，但清屏行为会调用 `cmd /c cls`

## 3. 安装与编译

### 3.1 从源码编译

```bash
go build -o sshtools .
```

编译完成后得到可执行文件 `sshtools`。

### 3.2 运行

```bash
./sshtools
```

如果需要显式指定配置文件：

```bash
./sshtools --config /path/to/config.yaml
```

## 4. 配置文件加载优先级

程序按以下优先级查找配置：

1. `--config` 指定的路径
2. 当前目录下的 `./config.yaml`
3. 用户目录下的 `~/.ssh-tool/config.yaml`
4. 二进制内嵌的 `default.yaml`

说明：

- 找到第一个可读配置后立即使用，不再继续查找
- 如果配置文件存在但解析失败，会直接报错退出
- 使用内嵌演示配置启动时，界面会提示当前正在使用默认配置
- 程序不会主动把配置写回磁盘

## 5. 配置文件示例

推荐使用 YAML。

```yaml
default_ip_type: public
ssh_timeout: 15s

groups:
  - name: 生产环境
    tag: prod
    machines:
      - name: web-server-01
        intranet_ip: 192.168.1.10
        public_ip: 1.2.3.4
        port: 22
        user: root
        private_key_path: ~/.ssh/id_rsa
        private_key_passphrase: ""

  - name: 测试环境
    tag: test
    machines:
      - name: staging-server-01
        intranet_ip: 192.168.2.10
        public_ip: 5.6.7.8
        port: 22
        user: ubuntu
        password: "your-password"

machines:
  - name: old-server-01
    intranet_ip: 192.168.9.10
    public_ip: ""
    port: 22
    user: root
    password: "your-password"
```

### 5.1 字段说明

| 字段 | 说明 |
|------|------|
| `default_ip_type` | 默认连接方式，推荐使用 `public` 或 `intranet` |
| `ssh_timeout` | SSH 超时时间，例如 `15s` |
| `groups[].name` | 分组显示名称 |
| `groups[].tag` | 分组唯一标识，用于 `p <tag>` |
| `machines[].name` | 主机名，全局唯一 |
| `intranet_ip` | 内网 IP，可为空 |
| `public_ip` | 外网 IP，可为空 |
| `port` | SSH 端口，默认 22 |
| `user` | 登录用户名 |
| `password` | 登录密码 |
| `private_key_path` | 私钥路径，支持 `~` 和相对配置目录 |
| `private_key_passphrase` | 私钥口令，可为空 |
| `platform` | 平台字段，用于列表展示，可为空 |

### 5.2 认证规则

- 同一主机优先使用 `private_key_path`
- 私钥不可用时，如果配置了 `password`，自动回退到密码登录
- `password` 和 `private_key_path` 至少配置一项
- 私钥路径支持 `~` 展开
- 相对路径以配置文件所在目录为基准

### 5.3 分组规则

- `groups` 和顶层 `machines` 可以共存
- 顶层 `machines` 会自动归入 `未分组 / default`
- 主机全局 ID 跨分组连续递增
- 搜索、连接、上传、下载都支持按全局 ID 操作

## 6. 启动后的主菜单

启动后会进入持续交互模式，提示符为：

```text
Opt>
```

常用命令如下：

| 命令 | 说明 |
|------|------|
| `主机名/IP关键词` | 模糊搜索并连接 |
| `/关键词` | 精确搜索并连接 |
| `数字 ID` | 按全局 ID 直接连接 |
| `p` | 显示所有主机 |
| `p <tag>` | 只显示指定分组 |
| `g` | 显示分组概览 |
| `u` | 上传文件到主机 |
| `d` | 从主机下载文件 |
| `r` | 重新加载配置 |
| `?` | 显示帮助 |
| `q` | 退出程序 |

## 7. 主机搜索与连接

### 7.1 直接连接

在 `Opt>` 下可以直接输入：

- 主机名片段
- 内网 IP 片段
- 外网 IP 片段
- 全局主机 ID

示例：

```text
Opt> nginx
Opt> /192.168.1.10
Opt> 3
```

行为说明：

- 只有一个匹配结果时，直接连接
- 有多个匹配结果时，进入结果选择页
- 如果配置了 `default_ip_type`，连接时不会再次询问公网/内网
- 如果未配置 `default_ip_type`，连接前会提示选择公网或内网地址

### 7.2 结果选择页操作

当搜索结果超过一条时，可使用以下输入：

| 输入 | 说明 |
|------|------|
| `1` / `2` / `3` | 按当前页显示 ID 连接 |
| `关键词` | 对当前结果重新模糊搜索 |
| `//关键词` | 在当前结果中继续二级过滤 |
| `/关键词` | 基于初始结果做精确搜索 |
| `n` | 下一页 |
| `b` | 上一页 |
| `q` | 退出当前页 |
| 回车 | 返回上一级 |

### 7.3 会话结束

远程主机中执行 `exit` 后，会自动回到主菜单并显示：

```text
已断开连接
```

## 8. 主机列表

### 8.1 查看全部主机

```text
Opt> p
```

列表页展示字段：

| 字段 | 说明 |
|------|------|
| `ID` | 当前结果集内的显示编号 |
| `名称` | 主机名称 |
| `内网 IP` | 内网地址 |
| `外网 IP` | 外网地址 |
| `端口` | SSH 端口 |
| `用户` | 登录用户 |
| `平台` | 平台信息 |

列表页支持：

- 输入显示 ID 直接连接
- 输入普通关键词进行模糊搜索
- 输入 `//关键词` 进行二级筛选
- 输入 `/关键词` 做精确搜索
- 输入 `n` / `b` 翻页
- 输入 `q` 或直接回车返回主菜单

### 8.2 查看指定分组

```text
Opt> p prod
```

如果分组不存在，会提示：

```text
未找到分组 "prod"，输入 g 查看所有分组
```

### 8.3 查看分组概览

```text
Opt> g
```

会显示：

- 分组序号
- 分组名称
- Tag
- 主机数量

## 9. 上传文件

命令：

```text
Opt> u
```

交互流程：

1. 输入主机名或全局 ID
2. 输入本地文件路径
3. 输入远程目标路径
4. 如果本地路径是目录，程序会询问是否递归上传
5. 显示上传进度条
6. 完成后输出远程保存路径

说明：

- 支持上传单个文件
- 支持目录递归上传
- 远程目标目录不存在时会自动创建

## 10. 下载文件

命令：

```text
Opt> d
```

交互流程：

1. 输入主机名或全局 ID
2. 输入远程文件或目录路径
3. 输入本地保存路径
4. 显示下载进度条
5. 完成后打印本地保存路径

说明：

- 本地保存路径直接回车时，默认保存到当前目录
- 支持下载目录
- 支持远程通配符路径，例如 `*.log`
- 如果本地目录不存在，会自动创建

## 11. 重新加载配置

命令：

```text
Opt> r
```

适用场景：

- 新增主机后不想重启程序
- 修改了 `default_ip_type`
- 调整了分组或认证信息

## 12. 清屏渲染规则

以下场景会自动清屏并从顶部重新渲染：

- 程序首次启动
- 输入 `p`
- 输入 `p <tag>`
- 列表页内继续搜索或翻页
- SSH 会话断开并返回主菜单

以下场景不会清屏：

- `g` 分组概览
- 连接提示信息
- SSH 交互会话本身
- 上传下载进度
- 普通错误和警告输出

## 13. 常见问题

### 13.1 没有找到配置文件怎么办？

程序会自动回退到内嵌演示配置。你可以创建以下任一文件：

- `./config.yaml`
- `~/.ssh-tool/config.yaml`

或者启动时使用：

```bash
./sshtools --config /your/config.yaml
```

### 13.2 私钥登录失败怎么办？

检查以下几点：

- `private_key_path` 是否存在
- 路径是否写对
- 私钥是否需要口令
- 如果私钥加密，是否配置了 `private_key_passphrase`

### 13.3 为什么连接时还会询问公网还是内网？

因为当前配置没有设置 `default_ip_type`。可在配置中增加：

```yaml
default_ip_type: public
```

或：

```yaml
default_ip_type: intranet
```

### 13.4 下载目录时本地路径如何决定？

- 如果直接回车，默认下载到当前目录
- 如果输入的是目录路径，会把远程目录保存到该目录下
- 如果输入的是不存在的目标目录，程序会自动创建
