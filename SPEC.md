## 二、配置文件

格式：YAML
依赖库：viper（支持 YAML / TOML / ENV 多格式）

---

### 2.1 配置文件读取优先级（从高到低）

1. --config 参数指定路径
2. ./config.yaml（二进制文件所在当前目录）
3. ~/.ssh-tool/config.yaml（用户家目录）
4. 内嵌 default.yaml（编译时打包进二进制的演示配置，兜底，不落盘）

规则：
- 按优先级依次检查，找到第一个可读配置立即使用，不再继续查找
- 配置文件存在但解析失败时，打印具体错误原因并退出，不继续查找下一级
- 使用内嵌配置启动时打印提示：
    "未找到配置文件，当前使用演示配置"
    "可创建 ./config.yaml 或 ~/.ssh-tool/config.yaml 以连接真实主机"
- 程序任何情况下不主动将配置写入磁盘

---

### 2.2 配置文件字段结构

default_ip_type: public       # 全局默认连接方式：public（外网）/ intranet（内网）
                              # 未配置时，每次连接前手动询问用户选择

groups:
  - name: 生产环境             # 分组显示名称
    tag: prod                 # 分组唯一标识，用于命令行过滤，英文无空格
    machines:
      - name: web-server-01   # 主机名，全局唯一标识
        intranet_ip: 192.168.1.10
        public_ip: 1.2.3.4    # 外网 IP，可为空字符串
        port: 22
        user: root
        password: "xxx"                    # 登录密码，与 private_key_path 至少配置一项
        private_key_path: ~/.ssh/id_rsa    # SSH 私钥路径，支持 ~ 展开和相对配置文件的路径
        private_key_passphrase: ""         # 私钥口令，无口令时留空

  - name: 测试环境
    tag: test
    machines:
      - name: staging-server-01
        intranet_ip: 192.168.2.10
        public_ip: 5.6.7.8
        port: 22
        user: ubuntu
        private_key_path: ~/.ssh/id_rsa

machines:                     # 顶层 machines（无分组），归入"未分组"展示
  - name: old-server-01
    intranet_ip: 192.168.9.10
    public_ip: ""
    port: 22
    user: root
    password: "xxx"

---

### 2.3 认证方式规则

优先级（同一主机）：
1. private_key_path 配置了且文件存在 → 使用私钥登录
2. 私钥不可用且 password 不为空 → 使用密码登录
3. 两者均未配置 → 启动时报错，提示该主机缺少认证信息，跳过该主机

私钥路径处理：
- 支持 ~ 展开为用户家目录
- 支持相对路径，基准为配置文件所在目录（非程序运行目录）
- 私钥文件不存在时打印警告，自动降级为密码登录

---

### 2.4 兼容规则

- 同一配置文件中 groups 和顶层 machines 可共存
- 顶层 machines 下的主机统一归入显示名称为"未分组"、tag 为 default 的分组
- 旧格式（只有顶层 machines，无 groups）完全兼容，不需要迁移

---

### 2.5 主机全局 ID 规则

- 所有分组的主机统一编号，ID 全局唯一，跨分组连续递增（从 1 开始）
- 编号顺序：groups 按配置文件顺序，每组内按 machines 顺序，顶层 machines 排在最后
- 搜索、连接、上传、下载均支持通过全局 ID 或主机名操作

## 三、启动方式与交互主循环

直接执行 `ssh-tool` 进入持续交互界面（类 JumpServer 风格），
使用 bufio 读取 stdin，打印 `Opt> ` 提示符，循环等待用户输入。

欢迎界面示例：

Welcome to SSH Tool

     1) Enter 输入部分 IP / 主机名 搜索并登录（唯一匹配时直接连接）
     2) Enter    输入 / 前缀 + IP / 主机名 精确搜索
     3) Enter    输入 p   显示所有主机列表
     3) Enter    输入 p   <tag>只显示指定分组，如：p prod
     4) Enter    输入 g   显示分组概览（分组名、tag、主机数量）
     5) Enter    输入 u   上传文件到指定主机
     6) Enter    输入 d   从指定主机下载文件
     7) Enter    输入 r   重新加载配置文件
     8) Enter    输入 ?   显示帮助
     9) Enter    输入 q   退出

Opt>


### 3.1 清屏渲染规则

以下操作执行前先清屏（输出 \033[H\033[2J），再从终端顶部开始渲染内容：
- 输入部分 IP / 主机名 搜索并登录（唯一匹配时直接连接）
- 输入 p（显示所有主机列表）
- 输入 p <tag>（显示指定分组列表）
- 在列表页内继续输入关键词搜索过滤
- SSH 会话断开后返回主菜单
- 程序首次启动展示欢迎界面

以下操作不清屏，在当前光标位置继续输出：
- 输入 g（显示分组概览）
- 连接主机时的提示信息（正在连接...）
- SSH 会话本身（会话接管终端，不干预）
- 上传 / 下载进度条
- 错误提示、警告信息

实现方式：
- 封装 clearScreen() 函数，写入 \033[H\033[2J 到 os.Stdout
- Linux / macOS 直接生效
- 如需兼容 Windows，检测 GOOS 后改为执行 cmd /c cls

### 4.2 连接主机

#### 触发方式

| 输入 | 行为 |
|------|------|
| 关键词（主机名或 IP 部分字符） | 跨所有分组模糊搜索 |
| / 前缀，如 /192.168.1.10 | 跨所有分组精确搜索 |
| 纯数字，如 3 | 直接按全局 ID 连接 |

#### 搜索结果展示（附带分组 tag）
[Host]> nginx
匹配结果：
ID | 名称             | 分组     | 内网 IP          | 外网 IP
----+------------------+----------+------------------+----------
1 | prod-nginx-1     | prod     | 192.168.1.10     | 1.2.3.4
2 | prod-nginx-2     | prod     | 192.168.1.11     | 1.2.3.5
3 | staging-nginx    | staging  | 192.168.2.10     | 5.6.7.8
页码：1，每页行数：35，总页数：1，总数量：3
提示：输入资产ID直接登录，二级搜索使用 // + 字段，如：//192 上一页：b 下一页：n
搜索：nginx
[Host]> 

行为逻辑：
- 唯一匹配时，自动连接，无需确认
- 在主菜单搜索进入匹配结果页后，继续输入关键词、//二级搜索或 /精确搜索，若结果唯一，则自动连接，无需再次输入 ID
- 在 p 或 p <tag> 列表页搜索后，若结果唯一，则自动连接，无需再次输入 ID
- 多个匹配时，展示列表，提示用户选择 ID
- 配置了 default_ip_type 时，自动使用公网或内网 IP；未配置时连接前询问使用 [1] 外网 IP 还是 [2] 内网 IP
- SSH 连接需申请 PTY 伪终端，将 os.Stdin/Stdout/Stderr
  直接绑定到 SSH session，实现完整交互式终端体验
- 会话退出后返回主交互菜单，打印"已断开连接"

依赖库：golang.org/x/crypto/ssh

---

### 4.3 上传文件（输入 u）

交互流程：
1. 提示输入主机名或 ID
2. 配置了 default_ip_type 时自动选择公网/内网 IP，否则提示选择
3. 提示输入本地文件路径
4. 提示输入远程目标路径
5. 展示上传进度条
6. 完成后返回主菜单

支持场景：
- 单个文件上传
- 目录递归上传（检测到目录时询问是否递归）

依赖库：github.com/pkg/sftp，github.com/schollz/progressbar

---

### 4.4 下载文件（输入 d）

交互流程：
1. 提示输入主机名或 ID
2. 配置了 default_ip_type 时自动选择公网/内网 IP，否则提示选择
3. 提示输入远程文件或目录路径
4. 提示输入本地保存路径（可直接回车使用默认值）
5. 展示下载进度条
6. 完成后打印保存路径，返回主菜单

---

#### 本地保存路径处理规则（重要）

| 用户输入 | 行为 |
|----------|------|
| 直接回车（空） | 使用当前目录（os.Getwd()），文件名取自远程路径的 path.Base() |
| 输入已存在的目录（以 / 结尾或 Stat 判断为目录） | 拼接远程文件名保存 |
| 输入不存在的目录 | 提示"目录不存在，是否创建？[y/N]"，y 则 os.MkdirAll 后保存，N 则重新输入 |
| 输入具体文件路径 | 按指定文件名保存，父目录不存在同样询问是否创建 |
| 文件已存在 | 提示是否覆盖，或自动重命名为 filename(1).ext |

支持场景：
- 单文件下载
- 目录递归下载（sftp.Stat 判断 IsDir()，使用 sftp.Walk() 遍历，保留目录结构）
- 通配符匹配（sftp.Glob(pattern) 展开后批量下载）

## 六、其他要求

- 使用 Go Modules 管理依赖

- SSH 连接超时默认 15 秒，可在配置文件中配置

## 七、配置文件加载优先级

使用 Go 1.16+ 的 embed 包将默认配置模板打包进二进制文件。

加载优先级（从高到低）：
1. ./config.yaml              — 二进制文件所在的当前目录
2. ~/.ssh-tool/config.yaml    — 用户家目录下的全局配置
3. 内嵌 default.yaml          — 编译时打包进二进制的默认配置（兜底）

行为规则：
- 按优先级依次检查，找到第一个可读的配置文件立即使用，不再继续查找
- 内嵌配置任何情况下不写入磁盘
- 使用内嵌配置启动时打印提示："当前使用默认配置"
- 配置文件存在但解析失败时，打印具体错误原因并退出，不继续查找下一级

安全原则：
- 程序任何情况下不主动将配置写入磁盘
- 用户密码仅存在于用户手动创建的配置文件中
- 建议用户对配置文件设置权限 chmod 600
